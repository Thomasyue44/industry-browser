# 产业链公司树状速查

这是从 `/Users/xurui/Documents/复盘/从产业角度的公司整理2026.xlsx` 生成的树状浏览工具。

## 怎么用

打开 `index.html`。

- 左侧查询公司名、代码或批注关键词。
- 如果同一家公司出现在多个子表，左侧只显示一张公司卡。
- 中间会显示这家公司出现过的所有子表树，命中分支会高亮，上级路径会浅色标出。
- 产业标题也可以展开或收起；未选公司时默认只显示产业标题，方便先扫全局。
- 有子分支的节点左侧有三角符号，点击该节点可展开或收起子分支。
- 点击“清除选择”后会进入未选公司状态，中间显示全部产业链，方便直接浏览各个产业。
- 未选公司时点击某个分支，右侧会列出该分支下每家公司的入选原因和 Excel 坐标。
- 右侧按子表分组显示公司为什么属于这些分支，说明来自 Excel 对应单元格的批注。

## 怎么更新

继续在原 Excel 里维护内容。改完后，运行 `refresh-data.command`，它会重新生成 `all-industries-data.js`。

## 怎么便捷维护

直接打开 `index.html` 或线上速查页。

- 点击右上角“维护模式”。
- 搜索公司后，在右侧点击“新增归属”，也可以编辑或删除已有归属。
- 新增时可以选择现有分支，也可以在任意父级下面新建分支。
- 修改完成后点击“保存到线上”。
- 第一次保存需要粘贴 GitHub fine-grained token；token 只保存在当前浏览器，不写进网页代码。
- 保存成功后，GitHub Pages 通常会在 1-3 分钟内更新线上网页。

## 当前抽取结果

- 抽取 20 个产业子表
- 分支来自每个子表数据区上方的层级表头
- 公司归属来自 `√` 等非空单元格
- 解释来自对应单元格批注
- 如果某个公司名等于另一张卡的“全称”，会合并到同一公司卡里

## 建议的长期结构

如果这个原型方向确认可用，后续可以把宽表背后的数据逐步整理成一张明细表：

`公司 | 代码 | 产业 | 一级环节 | 二级环节 | 三级环节 | 标记/说明 | 来源 | 更新时间`

这样可以同时生成宽表视图、公司卡片、产业链反查和相似公司，不会再被横向列数卡住。
